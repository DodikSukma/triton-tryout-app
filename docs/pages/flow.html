User Flow

Alur penggunaan platform untuk setiap role pengguna.

ℹ️ Semua role menggunakan halaman login yang sama (/login). Setelah login, sistem redirect otomatis ke dashboard sesuai role berdasarkan data session.

ADMIN

  • Login → /admin/dashboard
  • Sidebar: Kelola Guru → /admin/users?role=guru
  • Tambah Guru → Dialog form → POST /users
  • Toggle aktif/nonaktif guru → PATCH /users/:id/active
  • Edit profil guru → PUT /users/:id
  • Hapus guru → DELETE /users/:id
  • Sidebar: Kelola Siswa → /admin/users?role=siswa
  • Tambah Siswa (+ kelas) → POST /users
  • Profil Admin → /admin/profil

GURU

  • Login → /guru/dashboard
  • Lihat tryout milik sendiri → GET /tryouts
  • Buat Tryout → POST /tryouts → redirect ke soal builder
  • Tambah Soal Manual → POST /tryouts/:id/soal
  • Generate AI → pilih mapel → batch POST soal
  • Edit Soal → PUT /soal/:soalId
  • Hapus Soal → DELETE /soal/:soalId
  • Publish → PATCH /tryouts/:id/publish
  • Lihat Hasil → GET /hasil/rekap/:tryoutId
  • Profil Guru → /guru/profil

SISWA

  • Login → /siswa/dashboard
  • Lihat tryout → GET /tryouts/available
  • Cek status sesi → GET /sesi/by-tryout/:id
  • Konfirmasi → /siswa/tryout/:id
  • Mulai Tryout → POST /sesi → /exam/:sesiId
  • Lanjutkan → restore jawaban dari GET /sesi/:sesiId
  • Jawab soal (auto-save) → POST /sesi/:id/jawab
  • Kumpulkan / timer habis → POST /sesi/:id/selesai
  • Lihat Hasil → GET /hasil/:sesiId
  • Riwayat → GET /riwayat
  • Profil Siswa → /siswa/profil
📝
Detail Flow Ujian (Exam Page)
1

Siswa klik "Mulai Tryout"

Halaman konfirmasi /siswa/tryout/:id menampilkan info tryout, aturan, dan checkbox persetujuan.

2

POST /sesi → buat sesi baru

Server buat record sesi_tryout dengan status berlangsung. Redirect ke /exam/:sesiId.

3

Exam Page load

Fetch GET /tryouts/:id untuk data soal (is_benar disembunyikan untuk siswa). Fetch GET /sesi/:sesiId untuk restore jawaban yang sudah ada.

4

Siswa menjawab soal (auto-save)

Setiap kali siswa memilih opsi atau mengetik essay, langsung POST /sesi/:sesiId/jawab (upsert ON CONFLICT).

5

Timer habis atau klik Kumpulkan

Konfirmasi dialog → POST /sesi/:sesiId/selesai. Server fetch data soal dari soal-service (dengan role admin untuk mendapat is_benar), hitung nilai, simpan ke tabel hasil.

6

Redirect ke halaman hasil

Nilai, grade, statistik benar/salah/skip, dan breakdown per soal ditampilkan di /siswa/hasil/:sesiId.
