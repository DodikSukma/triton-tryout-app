Arsitektur Sistem

Browser (port 3000)
    │
    ▼
Next.js 14 Frontend
    │  axios + withCredentials: true
    ▼
API Gateway (port 4000)
    │  requireAuth → GET /auth/me → Redis lookup
    │  requireRole → inject X-User-Id, X-User-Role
    ├──► Auth Service    (port 4001)  →  db_auth    (users)
    ├──► User Service    (port 4002)  →  db_user    (profiles)
    ├──► Soal Service    (port 4003)  →  db_soal    (tryouts, soal, opsi)
    └──► Jawaban Service (port 4004)  →  db_jawaban (sesi, jawaban, hasil)

Cara Instalasi & Menjalankan

1 Prasyarat — Install dependensi sistem

Pastikan sudah terinstall:

  • Node.js v18+ — node -v
  • PostgreSQL 16 — psql --version
  • Redis — redis-cli ping harus mengembalikan PONG

# Jika belum ada Redis (macOS)

brew install redis

brew services start redis

2 Buat user & database PostgreSQL

# Masuk ke psql sebagai superuser

psql -U postgres

# Buat user

CREATE USER triton_user WITH PASSWORD 'triton_secret_2024';

# Buat 4 database

CREATE DATABASE db_auth OWNER triton_user;

CREATE DATABASE db_user OWNER triton_user;

CREATE DATABASE db_soal OWNER triton_user;

CREATE DATABASE db_jawaban OWNER triton_user;

\q

3 Inisialisasi skema tabel

psql -U triton_user -d db_auth -f services/auth-service/src/db/schema.sql

psql -U triton_user -d db_user -f services/user-service/src/db/schema.sql

psql -U triton_user -d db_soal -f services/soal-service/src/db/schema.sql

psql -U triton_user -d db_jawaban -f services/jawaban-service/src/db/schema.sql

4 Install & build semua service

# Install setiap service (jalankan dari root project)

for dir in services/auth-service services/user-service services/soal-service services/jawaban-service services/api-gateway; do

(cd "$dir" && npm install && npm run build)

done

# Install frontend

cd frontend && npm install

5 Jalankan semua service (5 terminal terpisah)

Buka 6 tab/terminal, jalankan masing-masing:

# Terminal 1 — Auth Service

cd services/auth-service

POSTGRES_HOST=localhost POSTGRES_DB=db_auth PORT=4001 npm start

# Terminal 2 — User Service

cd services/user-service

POSTGRES_HOST=localhost POSTGRES_DB=db_user PORT=4002 npm start

# Terminal 3 — Soal Service

cd services/soal-service

POSTGRES_HOST=localhost POSTGRES_DB=db_soal PORT=4003 npm start

# Terminal 4 — Jawaban Service

cd services/jawaban-service

POSTGRES_HOST=localhost POSTGRES_DB=db_jawaban PORT=4004 npm start

# Terminal 5 — API Gateway

cd services/api-gateway

AUTH_SERVICE_URL=http://localhost:4001 USER_SERVICE_URL=http://localhost:4002 SOAL_SERVICE_URL=http://localhost:4003 JAWABAN_SERVICE_URL=http://localhost:4004 PORT=4000 npm start

# Terminal 6 — Frontend

cd frontend && npm run dev

# Buka http://localhost:3000

6 Seed data awal (opsional tapi disarankan)

cd scripts && npm install

npx ts-node seed.ts

Membuat 8 akun: 1 admin, 2 guru, 5 siswa. Lihat Akun Default.

7 Tambahkan logo

Salin file logo ke:

cp /path/to/logo.png frontend/public/logo.png

Skema Database

db_auth— Autentikasi pengguna
CREATE TABLE users (
  id            UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email         VARCHAR(255) UNIQUE NOT NULL,
  password_hash VARCHAR(255) NOT NULL,
  role          VARCHAR(20) NOT NULL CHECK (role IN ('admin','guru','siswa')),
  is_active     BOOLEAN DEFAULT true,
  created_at    TIMESTAMPTZ DEFAULT NOW(),
  updated_at    TIMESTAMPTZ DEFAULT NOW()
);
db_user— Profil pengguna
CREATE TABLE profiles (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id        UUID UNIQUE NOT NULL,
  nama_lengkap   VARCHAR(255) NOT NULL,
  no_telepon     VARCHAR(20),
  kelas          VARCHAR(50),
  mata_pelajaran VARCHAR(255),
  avatar_url     TEXT,
  created_at     TIMESTAMPTZ DEFAULT NOW(),
  updated_at     TIMESTAMPTZ DEFAULT NOW()
);
db_soal— Tryout, soal, dan opsi
CREATE TABLE tryouts (
  id             UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  nama_tryout    VARCHAR(255) NOT NULL,
  mata_pelajaran VARCHAR(100) NOT NULL,
  durasi_menit   INTEGER NOT NULL DEFAULT 90,
  dibuat_oleh    UUID NOT NULL,
  status         VARCHAR(20) DEFAULT 'draft'
                 CHECK (status IN ('draft','published','closed')),
  created_at     TIMESTAMPTZ DEFAULT NOW(),
  updated_at     TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE soal (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tryout_id  UUID NOT NULL REFERENCES tryouts(id) ON DELETE CASCADE,
  nomor_soal INTEGER NOT NULL,
  tipe       VARCHAR(20) NOT NULL CHECK (tipe IN ('pilihan_ganda','essay')),
  pertanyaan TEXT NOT NULL,
  gambar_url TEXT,
  equation   TEXT,
  bobot      INTEGER NOT NULL DEFAULT 1,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE opsi_jawaban (
  id       UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  soal_id  UUID NOT NULL REFERENCES soal(id) ON DELETE CASCADE,
  huruf    CHAR(1) NOT NULL,
  teks     TEXT NOT NULL,
  is_benar BOOLEAN DEFAULT false
);
db_jawaban— Sesi, jawaban, dan hasil
CREATE TABLE sesi_tryout (
  id         UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  siswa_id   UUID NOT NULL,
  tryout_id  UUID NOT NULL,
  mulai_at   TIMESTAMPTZ DEFAULT NOW(),
  selesai_at TIMESTAMPTZ,
  status     VARCHAR(20) DEFAULT 'berlangsung'
             CHECK (status IN ('berlangsung','selesai','timeout')),
  UNIQUE(siswa_id, tryout_id)
);

CREATE TABLE jawaban (
  id           UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  sesi_id      UUID NOT NULL REFERENCES sesi_tryout(id),
  soal_id      UUID NOT NULL,
  jawaban_teks TEXT,
  opsi_id      UUID,
  created_at   TIMESTAMPTZ DEFAULT NOW(),
  updated_at   TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(sesi_id, soal_id)
);

CREATE TABLE hasil (
  id          UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  sesi_id     UUID UNIQUE NOT NULL REFERENCES sesi_tryout(id),
  siswa_id    UUID NOT NULL,
  tryout_id   UUID NOT NULL,
  total_benar INTEGER DEFAULT 0,
  total_soal  INTEGER NOT NULL,
  nilai       NUMERIC(5,2),
  dihitung_at TIMESTAMPTZ DEFAULT NOW()
);

API Reference

Semua request via frontend melalui API Gateway di http://localhost:4000. Kecuali disebutkan lain, semua endpoint butuh autentikasi.

Auth Service — port 4001

POST /auth/login Publik

Login dengan email dan password. Membuat sesi Redis dan set cookie triton.sid.

REQUEST BODY

{ "email": "admin@triton.id",
  "password": "admin123" }

RESPONSE 200

{ "success": true,
  "data": {
    "userId": "uuid",
    "role": "admin",
    "email": "admin@triton.id"
  }
}
POST /auth/logout Butuh sesi

Menghapus sesi Redis dan clear cookie.

GET /auth/me Butuh sesi

Cek sesi aktif. Digunakan oleh API Gateway untuk validasi setiap request.

RESPONSE 200

{ "success": true, "data": { "userId": "uuid", "role": "admin", "email": "..." } }
POST /auth/change-password Semua role

REQUEST BODY

{ "oldPassword": "...", "newPassword": "..." }

User Service — via Gateway /users

Method Path Role Keterangan
GET /users admin Daftar semua profil user
GET /users/profile/me semua Profil pengguna yang sedang login
PUT /users/profile/me semua Update profil sendiri
GET /users/:id admin Detail profil user berdasarkan ID
PUT /users/:id admin Update profil user
DELETE /users/:id admin Hapus profil user

Soal Service — via Gateway /tryouts, /soal

Method Path Role Keterangan
GET /tryouts guru Daftar tryout milik guru
POST /tryouts guru Buat tryout baru
GET /tryouts/:id guru Detail tryout + daftar soal
PUT /tryouts/:id guru Update info tryout
PATCH /tryouts/:id/publish guru Ubah status: draft/published/closed
DELETE /tryouts/:id guru Hapus tryout + semua soalnya
POST /tryouts/:id/soal guru Tambah soal ke tryout
PUT /soal/:soalId guru Edit soal
DELETE /soal/:soalId guru Hapus soal
GET /tryouts/available siswa Daftar tryout yang sudah published

Jawaban Service — via Gateway /sesi, /hasil

Method Path Role Keterangan
POST /sesi siswa Mulai sesi tryout (body: tryout_id)
GET /sesi/:sesiId siswa Detail sesi + semua jawaban
POST /sesi/:sesiId/jawab siswa Simpan/update jawaban satu soal
POST /sesi/:sesiId/selesai siswa Tandai tryout selesai
GET /hasil/:tryoutId siswa Hasil tryout milik siswa yang login
GET /hasil/rekap/:tryoutId guru Rekap nilai semua siswa untuk tryout ini

API Gateway — port 4000

Gateway memvalidasi sesi di setiap request terproteksi, lalu meneruskan ke service yang tepat dengan header tambahan:

HEADER YANG DIINJEKSIKAN

X-User-Id   → UUID pengguna
X-User-Role → admin | guru | siswa

ROUTING TABLE

/auth/*              → 4001 (publik)
/users/profile/me    → 4002 (semua)
/users/*             → 4002 (admin)
/tryouts/available   → 4003 (siswa)
/tryouts/* /soal/*   → 4003 (guru)
/sesi/*              → 4004 (siswa)
/hasil/*             → 4004 (siswa/guru)

Komponen Utama

Navbar

Sticky navbar untuk halaman publik. Logo, link navigasi, tombol "Masuk". Responsif dengan mobile hamburger menu.

Sidebar

Sidebar dashboard untuk semua role. Nav items role-spesifik, highlight item aktif, avatar pengguna, tombol logout.

RoleGuard

Wrapper komponen yang cek sesi via /auth/me. Redirect ke login jika tidak ada sesi, atau ke dashboard yang sesuai jika role salah.

api.ts

Axios instance dengan withCredentials: true dan interceptor yang redirect ke /login jika response 401.

Alur Autentikasi

  1. 1

    Pengguna submit form login

    Frontend POST /auth/login ke API Gateway → diteruskan ke Auth Service

  2. 2

    Auth Service verifikasi bcrypt

    Cek email di db_auth, bandingkan hash password dengan bcrypt

  3. 3

    Buat sesi Redis

    Simpan {"{userId, role, email}"} di Redis, set cookie triton.sid HttpOnly 8 jam

  4. 4

    Frontend redirect ke dashboard

    GET /auth/me → ambil role → router.push("/admin/dashboard")

  5. 5

    Setiap request selanjutnya

    Browser kirim cookie otomatis → Gateway validasi ke /auth/me → inject header X-User-Id dan X-User-Role

Fitur Lengkap

Admin

Dashboard Admin

Manajemen User

Lihat semua user (nama, kelas, telepon) dalam tabel yang dapat diurutkan.

Statistik Dashboard

Jumlah total user, tryout aktif, dan sesi yang sedang berlangsung.

Guru

Dashboard Guru

Buat Tryout

Form dengan nama tryout, mata pelajaran, dan durasi menit. Disimpan sebagai status "draft".

Kelola Soal

Tambah soal pilihan ganda/esai, lihat daftar soal, hapus soal dari tryout.

Publikasikan Tryout

Ubah status tryout dari "draft" → "published" agar dapat dikerjakan siswa, atau "closed" untuk menutup.

Rekap Nilai

Lihat rekap nilai semua siswa untuk setiap tryout via GET /hasil/rekap/:tryoutId.

Siswa

Dashboard Siswa

Pilih Tryout

Daftar tryout yang tersedia (status published), lengkap dengan mata pelajaran dan durasi.

Kerjakan Tryout

Navigasi soal per soal, pilih opsi jawaban, jawaban tersimpan otomatis ke server.

Selesaikan Tryout

Tombol "Selesai & Kumpulkan" di soal terakhir menandai sesi sebagai selesai.

Lihat Hasil

Hasil nilai tersimpan dan dapat dilihat kembali lewat endpoint /hasil/:tryoutId.
